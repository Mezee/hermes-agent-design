# Prototype To Production Checklist

Use this checklist before promoting a Hermes workflow to a more complex maturity level.

## Before Level 2

- the Level 1 workflow has been run against real work
- the user has corrected drift at least a few times
- the output quality is already acceptable
- the specialist boundary is obvious
- separate credentials or memory would help

## Before Level 3

- there are at least two meaningful specialists
- each specialist has a stable scope
- routing rules are understandable in plain language
- the orchestrator will simplify access instead of adding ceremony

## Before Level 4

- the workflow has survived repeated real runs
- error paths are known
- approvals are explicit
- completion is signaled explicitly
- the team can resume from checkpoints or task state
- the user is not relying on hidden manual cleanup

## Review Heuristic

If the current level still needs heavy coaching, keep it there.

If the architecture feels bigger than the business problem, collapse back down a level.
